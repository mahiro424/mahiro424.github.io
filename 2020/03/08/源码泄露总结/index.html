源码泄露总结

网站源码泄露分类

目录遍历

由于配置错误导致网站的目录可被遍历，一般该类漏洞可以为后续利用提供一些信息上的帮助。

PHPINFO

phpinfo() 是php中查看相关信息的函数，当在页面中执行phpinfo()函数时，php会将自身的所有信息全部打印出来。在phpinfo中会泄露很多服务端的一些信息。

备份文件下载

网站源码

常用网站源码备份文件后缀：
tar,tar.gz,zip,rar,bak,swp,txt,html
常用网站备份文件名：
web,website,backup,back,www,wwwroot,temp
利用工具：
dirmap,dirsearch,御剑等

.bak文件

有些时候网站管理员可能为了方便，会在修改某个文件的时候先复制一份，将其命名为【xxx.bak】。而大部分Web Server对bak文件并不做任何处理，导致可以直接下载，从而获取到网站某个文件的源代码
利用工具：
dirmap,dirsearch,御剑等扫到后
curl http://example.com/example.php.bak

vim缓存

在使用vim时会创建临时缓存文件，关闭vim时缓存文件则会被删除，当vim异常退出后，因为未处理缓存文件，导致可以通过缓存文件恢复原始文件内容以example.php为例：第一次产生的交换文件名为【.example.php.swp】再次意外退出后，将会产生名为 【.example.php.swo】 的交换文件第三次产生的交换文件则为 【.example.php.swn】
利用工具：
dirmap,dirsearch,御剑等扫到后
curl http:example.com/.example.php.swp
或者
wget http:example.com/.example.php.swp
wim index.php 选择R恢复文件

.DS_Store

.DS_Store文件利用.DS_Store是Mac OS保存文件夹的自定义属性的隐藏文件。通过.DS_Store可以知道这个目录里面所有文件的清单。
利用工具：
dirmap,dirsearch,御剑等扫到后
curl  http:example.com/.DS_Store
或者用Python .DS_Store parser (https://github.com/gehaxelt/Python-dsstore))
wget http:example.com/.DS_Store -q0 example/.DS_Store
python main.py example/.DS_Store

Git泄露

Log

查看历史版本
利用工具：
dirmap,dirsearch,御剑等扫到后
python GitHack.py http:example.com/.git/
git log
第一种方法：
git diff 该历史版本的前五位字符
或者
git diff HEAD^
第二种方法
git reset --hard 该历史版本的前五位字符
或者
git reset --hard HEAD^

Stash

git栈
利用工具：
dirmap,dirsearch,御剑等扫到后
python GitHack.py http:example.com/.git/
第一种方法：
git stash list
git stash pop/git stash apply
第二种方法：
git stash list
git stash show
cat .git/refs/stash
it diff 该历史版本的前五位字符

Index

index暂存区文件
利用工具：
dirmap,dirsearch,御剑等扫到后
python GitHack.py http:example.com/.git/
git ls-files
git ls-files -s //查看Blog对象
git cat-file -p 该文件Blog对象的前五位字符 //通过Blog对象查案该文件

SVN泄露

当svn使用了checkout命令后就会生成.svn文件，里面存储着备份信息。svn信息泄露漏洞主要利用了里面的entrist文件，通过.svn/entrist可以下载里面的所有代码，但是只能作用在svn1.6之前的版本；第二个是作用在svn1.7后的版本，svn1.7后的版本引入一个名为wc.db的数据库数据存放文件来管理文件，通过访问.svn/wc.db可以下载到本地
利用工具：
dirmap,dirsearch,御剑等扫到后，利用dvcs-ripper(https://github.com/kost/dvcs-ripper)
perl rip-svn.pl -u http://example.com/.svn/ (kali自带，其他的linux需要下载perl和svn环境)
ls -al  //.svn是隐藏文件夹要用-al才能看到
然后查看wc.db数据库或者进入缓存文件夹pristine文件夹中查看

HG泄露

HG init的时候会生成.hg
利用工具：
dirmap,dirsearch,御剑等扫到后，利用dvcs-ripper(https://github.com/kost/dvcs-ripper)
perl rip-hg.pl -u http://example.com/.hg/ (kali自带，其他的linux需要下载perl和svn环境)
ls -al  //.hg是隐藏文件夹要用-al才能看到
然后进入.hg/store/fncache用curl http://example.com/.hg/store/data/example.xxx.i或者curl http://example.com/example.xxx

BZR泄露

利用工具：
dirmap,dirsearch,御剑等扫到后，利用dvcs-ripper(https://github.com/kost/dvcs-ripper)
perl rip-bzr.pl -u http://example.com/.bzr/ (kali自带，其他的linux需要下载perl和svn环境)
ls -al  //.hg是隐藏文件夹要用-al才能看到

WEB-INF/web.xml泄露

WEB-INF主要包含一下文件或目录：
/WEB-INF/web.xml：Web应用程序配置文件，描述了 servlet 和其他的应用组件配置及命名规则
/WEB-INF/classes/：含了站点所有用的 class 文件，包括 servlet class 和非servlet class，他们不能包含在 .jar文件中
/WEB-INF/lib/：存放web应用需要的各种JAR文件，放置仅在这个应用中要求使用的jar文件,如数据库驱动jar文件
/WEB-INF/src/：源码目录，按照包名结构放置各个java文件
/WEB-INF/database.properties：数据库配置文件
利用方式：
通过找到web.xml文件，推断class文件的路径，最后直接class文件，在通过反编译class文件，得到网站源码。
