什么是JWT令牌
Json web token (JWT), 是为了在网络应用环境间传递声明而执行的一种基于JSON的开放标准（(RFC 7519).该token被设计为紧凑且安全的，特别适用于分布式站点的单点登录（SSO）场景。JWT的声明一般被用来在身份提供者和服务提供者间传递被认证的用户身份信息，以便于从资源服务器获取资源，也可以增加一些额外的其它业务逻辑所必须的声明信息，该token也可直接被用于认证，也可被加密。
JWT令牌的形式
eyJhbGci0iJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIi0iIxMjM0NTY30DkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ
.SflKxwRJSMeKKF2QT4fwpMeJf36P0k6yJV_adQssw5c
实际上是上面这一段这样的数据，这段数据由三个部分组成：

Header

  • eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9解码为{“alg”:”HS256”,”typ”:”JWT”},alg属性表示签名的算法（algorithm），默认是 HMAC SHA256（写成 HS256）；typ属性表示这个令牌（token）的类型（type），JWT 令牌统一写为JWT。

Payload

  • eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ解码为{“sub”:”1234567890”,”name”:”John Doe”,”iat”:1516239022}
    JWT 规定了7个官方字段，供选用
    iss (issuer)：签发人
    exp (expiration time)：过期时间
    sub (subject)：主题
    aud (audience)：受众
    nbf (Not Before)：生效时间
    iat (Issued At)：签发时间
    jti (JWT ID)：编号

Signature

  • Signature 部分是对前两部分的签名，防止数据篡改。首先，需要指定一个密钥（secret）。这个密钥只有服务器才知道，不能泄露给用户。然后，使用 Header 里面指定的签名算法（默认是 HMAC SHA256），按照下面的公式产生签名。算出签名以后，把 Header、Payload、Signature 三个部分拼成一个字符串，每个部分之间用”点”（.）分隔，就可以返回给用户。SflKxwRJSMeKKF2QT4fwpMeJf36P0k6yJV_adQssw5c解码为HMACSHA256( base64UrlEncode(header+”.”+base64UrlEncode(payload),密钥)，如果密钥为none或者undefined就不会有Signature字段

JWT安全问题一般有以下

  1. 修改算法为none
  2. 修改算法从RS256到HS256
  3. 信息泄漏 密钥泄漏
  4. 爆破密钥

  8. 登录模块接受传入的 username 和 password，然后从令牌的信息段中取出 secretid，从全局数组 secrets 中取出 secret，然后进行验证，验证通过之后置 session 中的 username 为登录时使用的 username

BabyUpload

0x01

考察知识点：PHP代码审计，session伪造

0x02

  1. 打开题目看到源码(太长了，直接put)
  2. 首先将session的存储的路径设为/var/babyctf，同时包含flag文件
  3. 判断session里面的username参数是否是admin，如果是在判断文件success.txt是否存在，都是的话就把success.txt删了然后显示flag的内容，都不是的话就把session里的username参数设置成guest
  4. 需要POST两个参数，分别是direction和attr，attr会被直接拼接在/var/babyctf这个路径后面，如果attr为private就把用户名继续拼接在后面，direction是选择上传或者下载的
  5. 如果direction的参数为upload，就把文件名拼接在后面，同时加上下划线和这个文件内容的sha256摘要值，即var/babyctf/$_POST[‘attr’]/文件名_sha256摘要值，然后判断是否有路径穿越，逐级创建目录，将文件储存到后面
  6. 如果direction的参数为download，就把文件名拼接到后面，再判断是否有路径穿越，然后将文件内容返回
  7. 那么解题思路就是：伪造session使自己变成admin->创建一个success.txt文件->读取flag，因为拼接文件名的方式是文件名_sha256摘要值，所以可以上传一个sess，内容是我们伪造的session内容，然后计算出sess文件的shaa256的摘要值，最后将cookie中的PHPSESSID换成这个摘要值，就可以伪造session，所以先要读取一下session文件里的内容，就要先知道源PHPSESSID，打开network查看
  8. 然后用download的方式查看session文件里存储的内容
  9. 然后将seesion的内容复制下来，将guest换成admin，保存成sess文件，计算sess文件的sha256摘要值
  10. 然后把sess文件用upload方式传到服务器上
  11. 然后构造success.txt文件，把attr置为success.txt，通过upload的方式传到服务器上
  12. 通过download方式，filename置为sess_sha256摘要值，读取session的内容，可以看到已经变为admin了
  13. 然后把cookie换成sha256摘要值，成功获得flag
